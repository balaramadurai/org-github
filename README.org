* org-github

GitHub Issues/PRs integration with Org-mode, featuring bidirectional state
sync, time tracking, and an optional team dashboard.

** Features

- Fetch GitHub issues and PRs into Org headings with full metadata
- Bidirectional sync: change TODO state in Org and it updates GitHub
- Clock in/out on issues with optional time logging to GitHub
- Create issues from Org headings
- Add comments to issues/PRs from Emacs
- Distinguish merged vs closed-without-merge PRs (DONE vs CANCELLED)
- PARA-compatible file organization via repo-to-file mappings
- *Optional team dashboard* with per-assignee progress bars and filtering

** Requirements

- Emacs 27.1 or later
- Org-mode 9.0 or later
- [[https://cli.github.com/][GitHub CLI (gh)]] installed and authenticated

** Installation

*** MELPA

(use-package org-github

** Configuration

*** Basic Setup

#+begin_src emacs-lisp
;; Default org file (defaults to github.org in org-directory)
(setq org-github-org-file "~/org/github.org")

;; Repos to track
(setq org-github-default-repos '("owner/repo1" "owner/repo2"))

;; Enable bidirectional sync (Org TODO changes update GitHub)
(org-github-mode 1)
#+end_src

*** PARA-Style File Mapping

Map repositories to specific files and parent headings:

#+begin_src emacs-lisp
(setq org-github-repo-file-alist
      '(("owner/project-repo" . ("~/org/projects.org" . "PROJECT MyProject"))
        ("owner/area-repo" . ("~/org/areas.org" . "TODO Work Tasks"))))
#+end_src

Issues/PRs will be placed under:
: * PROJECT MyProject
: ** GitHub Issues
: *** TODO #1 Issue title

*** PR State Configuration

Distinguish merged PRs from rejected ones:

#+begin_src emacs-lisp
;; Default: merged PRs → DONE, closed-without-merge → CANCELLED
(setq org-github-pr-merged-state "DONE")
(setq org-github-pr-closed-state "CANCELLED")
#+end_src

*** Time Tracking

#+begin_src emacs-lisp
(setq org-github-auto-clock t)          ; auto clock in
(setq org-github-log-to-github t)       ; post time as GitHub comment
#+end_src

** Usage

*** Download Issues/PRs

| Command                          | Description                     |
|----------------------------------+---------------------------------|
| =M-x org-github-download-issues=  | Download open issues from repo  |
| =M-x org-github-download-prs=     | Download open PRs from repo     |
| =M-x org-github-download-all=     | Download both issues and PRs    |
| =M-x org-github-sync-repos=       | Sync all default repos          |
| =M-x org-github-full-sync=        | Full sync (new + update states) |

*** Bidirectional Sync

When =org-github-mode= is enabled, changing a TODO state on a GitHub-linked
heading automatically updates the issue on GitHub:

- Mark DONE → closes the issue/PR (with optional comment)
- Reopen (TODO/NEXT from DONE) → reopens on GitHub
- Sub-states like NEXT or WAITING are preserved during sync

*** Time Tracking

| Command                            | Description                         |
|------------------------------------+-------------------------------------|
| =M-x org-github-clock-in=           | Clock in on current issue/PR        |
| =M-x org-github-clock-out-and-log=  | Clock out, optionally log to GitHub |

*** Issue Management

| Command                         | Description                     |
|---------------------------------+---------------------------------|
| =M-x org-github-view-issue=      | View issue/PR details in buffer |
| =M-x org-github-open-in-browser= | Open issue/PR in browser        |
| =M-x org-github-close-issue=     | Close issue/PR on GitHub        |
| =M-x org-github-add-comment=     | Add comment to issue/PR         |
| =M-x org-github-create-issue=    | Create issue from Org heading   |

** Team Dashboard (Optional)

The =org-github-dashboard= package provides an interactive team dashboard
built on =org-ql= and =org-super-agenda=.

*** Installation

#+begin_src emacs-lisp
(use-package org-github-dashboard
  :after org-github
  :commands (org-github-dashboard)
  :bind ("<f4>" . org-github-dashboard))
#+end_src

*** Features

- Per-assignee sections with open/done issue counts
- Progress bars showing team completion rates
- Interactive filtering by repo, assignee, status, and time period
- Inline sync of individual items from the dashboard
- Excluded assignees (for departed team members)

*** Dashboard Keybindings

| Key | Action                                   |
|-----+------------------------------------------|
| =/=   | Filter by repos, assignees, status, period |
| =S=   | Sync the item at point from GitHub       |

*** Configuration

#+begin_src emacs-lisp
;; Exclude departed team members
(setq org-github-dashboard-excluded-assignees '("former-dev"))

;; Hide assignees with zero items
(setq org-github-dashboard-hide-empty t)
#+end_src

** Issue Format

Issues are stored as Org headings with properties:

#+begin_example
*** TODO #42 Fix login bug :bug:enhancement:
:PROPERTIES:
:ISSUE_NUMBER: 42
:REPO: owner/repo
:STATE: open
:URL: https://github.com/owner/repo/issues/42
:AUTHOR: username
:CREATED_AT: [2025-01-15 Wed 10:30]
:UPDATED_AT: [2025-01-20 Mon 14:22]
:ASSIGNEES: user1, user2
:MILESTONE: v1.0
:END:

Issue body text goes here...
#+end_example
